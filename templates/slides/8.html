Python REST Prototype

Using Python as a Triple Pattern Fragments server is relatively simple. A web client queries the linked-data fragments server with url and one or more parameters in the query string.

  • s parameter for the subject
  • p parameter for the predicate
  • o parameter for the object

The Python server demonstrated today is built using the Falcon framework for quick development of fast REST services in Python. The SPARQL look-up in case of zero hits in the Redis cache is implemented but only lightly tested.

What about the GO server?

Due to multiple factors (like finishing a book), the Go language server is still in active development and not ready for prime time. Because the Redis cache is the primary source of delays and not the web server, the development of the Go server was not as a high priority.
